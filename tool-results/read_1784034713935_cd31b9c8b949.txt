     7→// DATABASE PROVIDER — SQLite (dev) ↔ PostgreSQL (prod)
     8→// ───────────────────────────────────────────────────────────────────
     9→// The SAME schema compiles against either provider. To switch:
    10→//
    11→//   1. Edit the `datasource db` block below:
    12→//        dev :  provider = "sqlite"      // DATABASE_URL=file:./db/custom.db
    13→//        prod:  provider = "postgresql"  // DATABASE_URL=postgresql://...
    14→//   2. Set `DATABASE_URL` accordingly (see `.env.example`).
    15→//   3. Apply migrations:
    16→//        dev :  bun run db:push
    17→//        prod:  bun run db:migrate        // prisma migrate deploy
    18→//   4. Seed:  bunx prisma db seed
    19→//   5. SQLite → PostgreSQL data migration: use `pgloader` or a one-off
    20→//      `prisma data proxy` snapshot (see PRODUCTION-CHECKLIST.md →
    21→//      "PostgreSQL Migration" for the full step-by-step).
    22→//
    23→// Notes:
    24→//   - SQLite has no native enums/arrays — we use String + constants in code.
    25→//   - PostgreSQL-specific DDL (RLS policies, pgvector) lives in
    26→//     `prisma/migrations/1_postgres_indexes/migration.sql` — only applied
    27→//     when the provider is `postgresql`.
