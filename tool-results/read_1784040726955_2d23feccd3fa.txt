  5740→**Top 3 things to fix first (P0):**

  5817→### What's already good (no action needed)
  5818→- ✅ **Webhook idempotency** — Stripe/PayU/MercadoPago/Wompi/Meta/WhatsApp all use `isDuplicateWebhook(generateWebhookId(rawBody, signature))` with 5-min TTL. Webhook signature verification via adapter-specific `webhookVerify`. Always return 200 to avoid provider retries.

  5820→- ✅ **Discriminated unions for multi-action POST** — `trafficker`, `remarketing`, `novedades`, `redelivery`, `wallet`, `marketplace`, `notifications` all use `z.discriminatedUnion('action', [...])` with TS-narrowed per-branch handlers. Excellent pattern.
  5821→- ✅ **HTTP method conventions** — GET for reads (no side effects), POST for creates, PATCH for partial updates, DELETE for removals. No GET-with-body anti-patterns. PUT is used only on `/api/integrations/credentials` (returns the registry — arguably should be GET).
  5822→- ✅ **Security headers** — middleware sets `X-Frame-Options: DENY`, `X-Content-Type-Options: nosniff`, `Strict-Transport-Security`, `Referrer-Policy`, `Permissions-Policy`, and `Content-Security-Policy: default-src 'none'` on JSON responses. Solid baseline.
