Seguridad 2FA (TOTP)

  1. Descarga Google Authenticator o Authy
  2. En Wallet → "Activar 2FA"
  3. Escanea el código QR con la app
  4. Guarda los 10 códigos de respaldo en un lugar seguro
  5. Cada retiro requiere el código de 6 dígitos

⚠ Importante

Los códigos de respaldo son de un solo uso. Si los agotas, debes reactivar 2FA. Los secretos están encriptados con AES-256-GCM.

Proceso de Retiro

1. Solicitas con monto + código 2FA
2. Estado: pending_2fa
3. Admin/Finance aprueba
4. Estado: processed · Balance descontado

Ley 2573 de 2026

Suplantación de identidad. KYC gate para crédito/cuotas. Verificación trazable con evidence hash. Vigencia 90 días.

✅ Implementado
🔐

Ley 1581 de 2012

Protección de datos. Consent records por propósito. DSR (acceso/borrado/portabilidad). Retención automática diaria.

✅ Implementado

Ley 1480 Art 47

Derecho al retracto. 5 días para cancelar. Reembolso automático vía pasarela. Plazo legal: 30 días.

✅ Implementado

Ley 1098 de 2006

Infancia y adolescencia. Age gate + consentimiento parental. Bloqueo de checkout para menores sin autorización.

✅ Implementado

Facturación electrónica. CUFE (SHA-384). Integración Alegra. Envío automático a DIAN + email al cliente.

Módulos 10-16

Monetización · Logística · Marketplace · Novedades · Gobernanza · Configuración

💵 Monetización

Comisión escalonada sobre GMV: 4.5% (inicial) → 3.0% (medio) → 1.75% (premium). Reconocimiento en 2 momentos: 50% datos completados + 50% despachado. Factura DIAN con CUFE.

🚚 Inteligencia Logística

Scores de clientes (0-100, VIP/Regular/En riesgo/Nuevo). Scores de transportadoras (on-time rate). Guías estancadas >3 días. Alertas de comportamiento.

🏪 Marketplace Cross-brand

Catálogo unificado de las 4 marcas. Referir leads entre marcas con comisión. Configurar compartir leads + porcentaje.

⚠️ Novedades

Sistema de incidencias post-venta. Casos NV-YYYY-XXXXX. Reintentos de entrega. Historial con mensajes + evidencia fotográfica.

🛡️ Gobernanza

Escalaciones pendientes (aprobar/rechazar). Decisiones de IA con confidence + revisión humana. Reglas: orden >$5M, primera compra, cambio de pago. Responsabilidad: merchant vs agent_provider.

⚙️ Configuración

Estrategia de pago por canal (Anticipado/COD/Híbrido). Umbrales del trafficker. Credenciales de canales. Webhooks & endpoints. Integraciones con estado real.

Solución de Problemas

Problemas Comunes y Soluciones

🔑 No puedo iniciar sesión

• Credenciales incorrectas → usa botones de demo
• Página en blanco → limpia cache (Ctrl+Shift+R)
• Redirect loop → borra cookies del sitio
• "Error crítico" → contacta soporte@ziay.co

📊 No veo datos

• Todo en cero → verifica tenant seleccionado
• Skeleton infinito → click "Refrescar"
• "Failed to load" → click "Reintentar"
• Datos antiguos → click "Refrescar"

🤖 La IA no responde

• Sin respuesta → verifica orquestador no en pausa
• Respuesta tardía → timeout 15s, usa fallback
• Respuesta incorrecta → confidence <0.6, escala a humano
• "Presupuesto excedido" → admin aumenta en Costos IA

💬 WhatsApp no recibe

• No llegan mensajes → verifica webhook en Meta Business
• Mensajes duplicados → 3 capas de deduplicación, verifica logs
• No se pueden enviar → verifica token Cloud API

👛 No puedo retirar

• "2FA requerido" → ingresa código TOTP
• "2FA inválido" → verifica código actual (30s)
• 2FA no configurado → activa en Wallet
• "Pending processing" → espera aprobación admin

💳 Pago no procesa

• Error en checkout → verifica credenciales en Integraciones
• Webhook no llega → verifica URL en plataforma
• Pago duplicado → sistema idempotente, verifica Pedidos
• PSE/PIX no disponible → verifica credenciales .env
