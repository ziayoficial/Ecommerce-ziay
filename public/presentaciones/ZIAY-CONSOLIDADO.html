Anti-fraude transaccional

Protección en cada pago

6 capas que detectan fraude antes de que el chargeback llegue

⚡
Velocity checks

Rate-limit por customer / IP / BIN de tarjeta. Un mismo dispositivo no puede hacer 100 pagos en 1 minuto.

🚫
Blocklist dinámica

Lista de clientes bloqueados, IPs, BINs y emails. Una devolución de chargeback alimenta la lista automáticamente.

🏛️
OFAC screening

Check contra listas restrictivas (OFAC SDN, ONU, Lista Clinton) para expansión a USA. Actualización semanal.

🔐
3DS / SCA

Stripe request_three_d_secure: 'any'. Obligatorio en Brasil (BACEN) y Europa (PSD2). Friction adicional en high-risk.

💳
CVV / AVS check

Validación explícita de cvv_result_code y avs_result_code. No confía solo en status: approved del gateway.

🔁
Chargeback loop

Endpoint POST /webhooks/chargeback recibe notificaciones del gateway y marca la orden + alimenta el blocklist. Loop cerrado.

Circuit Breaker

Resiliencia production-grade

Cuando el LLM o un gateway falla, ZIAY no se cae — se degrada con elegancia

🟢
Closed (normal)

Todo fluye. Cada llamada cuenta su latencia y resultado. Si >50% falla en 30s, el breaker salta a open.

🔴
Open (trip)

Las llamadas fallan rápido (fail-fast) sin tocar el LLM/gateway. Se devuelve respuesta fallback por agente (canned reply).

🟡
Half-open (probe)

Tras 30s, deja pasar 1 llamada de prueba. Si éxito → vuelve a closed. Si falla → vuelve a open. Auto-recuperación.

🔁
Retry + backoff

3 retries con exponential backoff (500ms → 5s) + ±20% jitter. 4xx (excepto 429) nunca se reintentan.

📉
Model fallback

Cadena glm-4.6-plus → glm-4.6 → glm-4.6-flash. Dos capas: retry dentro del modelo, fallback entre modelos.

📊
Dashboard + alerts

Estado del breaker en tiempo real. Alertas a Slack/WhatsApp cuando un breaker trippea. Trazabilidad en DecisionLog.

Handoff humano

La IA vende, el humano cierra lo difícil

El 17% de compradores confía en completar compras con IA. Para el 83% restante, el handoff es crítico.

⏸️
Pause bot

El operador pausa el bot desde el header con un click. Campo botEnabled=false + pausedReason. La IA deja de responder; el humano toma el control del chat.

🏷️
Badge en lista

Las conversaciones pausadas muestran un badge amarillo "Humano" en la bandeja de entrada. Filtro dedicado ?filter=paused para ver solo las que te tocan.

🤝
Handoff button

Botón visible en el header de cada conversación. Click → pausa + asigna al operador logueado + notifica por socket al equipo. Reanudar con un click cuando el humano termine.

🚨
Auto-escalada

Si sentiment.frustration=high o el cliente pide explícitamente "humano", el bot sugiere el handoff al operador. No requiere que el cliente sepa que existe.

📜
Contexto completo

El operador recibe el timeline completo de la conversación + perfil + historial de compras. No empieza de cero — retoma donde el bot lo dejó.

⚖️
Compliance ToS WhatsApp

El handoff asegura "human in the loop" — requisito para operar como BSP bajo el programa de partners de Meta (Jan 2026 ToS).
