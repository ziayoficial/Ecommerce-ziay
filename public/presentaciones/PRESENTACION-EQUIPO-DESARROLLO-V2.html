¿Qué es ZIAY?
Una plataforma de comercio conversacional multi-tenant

En una frase: es un sistema que permite a muchas tiendas vender por WhatsApp (y otros canales) usando un equipo de agentes de IA que atienden, cotizan, cobran y despachan — cada tienda con sus datos aislados, su catálogo y su marca.

💬 Conversacional

El cliente no llena formularios. Habla por WhatsApp como hablaría con un vendedor humano. La IA lleva la conversación.

🤖 Agentes IA

27 agentes especializados (perfilador, cotizador, cobrador, despachador…) coordinados por un Governor supervisor.

🏢 Multi-tenant

Cada tienda (tenant) tiene su catálogo, sus clientes, sus pagos y sus configuraciones. Aislados, sin filtraciones.

🧠 La analogía

Imagina un centro comercial con 500 tiendas. En cada una hay un vendedor experto disponible 24/7 que nunca se enferma, conoce el catálogo de memoria, calcula precios al instante y aprende qué argumentos venden más. ZIAY es ese centro comercial, en una sola plataforma.

Flujo principal05 / 22

Del anuncio a la entrega: el flujo completo

Una venta en ZIAY pasa por 6 etapas. Cada una tiene dueño, métrica y punto de control. Si algo falla, el sistema lo detecta y lo escalona.

1

Ad

El trafficker lanza un anuncio en Meta/Google/TikTok con un parámetro click_id. El cliente hace clic y llega al WhatsApp del merchant.

2

WhatsApp

El cliente escribe. El webhook de WhatsApp Cloud API recibe el mensaje y lo pasa a ZIAY con el click_id (asociación al anuncio).

3

9 agentes

El pipeline de 9 pasos se ejecuta: saludo → perfil → necesidad → catálogo → cotiza → objeciones → checkout → logística → post-venta.

4

Venta

El cliente paga con su método nativo (PSE, PIX, OXXO, tarjeta…). El dinero entra al merchant y la orden se confirma.

5

Logística

Se genera guía con el carrier elegido (Dropi, 99envios, Aveonline). El paquete se despacha y se trackea.

6

Post-venta

Tracking, novedades, re-entrega, reseña. La IA cuida al cliente hasta que recibe y queda feliz.

🧠 Lo importante

Cada etapa deja datos trazables. Podemos decir exactamente qué anuncio trajo qué conversación, qué paso del pipeline falló, qué carrier se demoró, qué método de pago abandonó el cliente. Sin trazabilidad no hay optimización — y ZIAY traza todo.

Multi-tenancy06 / 22

Cada tienda es un tenant aislado
con sus propios datos

En ZIAY conviven cientos de tiendas, pero cada una opera como si estuviera sola. Catálogos, clientes, conversaciones y configuraciones no se cruzan nunca — aislamiento lógico garantizado a nivel de dato y de infraestructura.

🔒 Qué se aísla por tenant

RecursoAislamiento
Catálogo de productosPor tenant
Conversaciones / mensajesPor tenant
Clientes / PIIPor tenant
Órdenes y pagosPor tenant
Configuración de agentesPor tenant
Cuentas de WhatsApp BusinessPor tenant
Integraciones (Shopify, etc.)Por tenant
Analytics y métricasPor tenant

⚙ Cómo se logra

1

Cada tabla tiene tenantId como columna obligatoria.

2

Todas las queries se filtran por tenantId en el ORM (Prisma).

3

El JWT del merchant incluye el tenantId — no se puede suplantar.

4

Variables de entorno (tokens de WhatsApp, API keys de pagos) por tenant, cifradas en reposo.

5

Logs y observabilidad etiquetados por tenant — sin cruzar datos en dashboards.

6

Rate limiting por tenant — un pico en una tienda no afecta a las demás.

⚠ Regla de oro

Nunca, bajo ninguna circunstancia, una query debe ejecutarse sin filtro de tenantId. Un solo query sin filtro puede exponer datos de otra tienda — y eso es causa inmediata de terminación de contrato y demanda. Es la regla más vigilada del sistema.

El Governor: valida todo

Cada agente puede equivocarse. El Governor no los deja llegar al cliente con el error. Es la capa de gobernanza que asegura calidad, cumplimiento legal y experiencia consistente — en cada mensaje, en cada tienda.

🧠 El modelo de confianza

Cada respuesta de cada agente tiene un score de confianza (0-1). Si es <0.6, no se envía sola — se escala a humano con todo el contexto. Esto evita alucinaciones en producción.

Confianza 0.95 — envía directo95%
Confianza 0.75 — envía + log75%
Confianza 0.55 — escala humano55%

El sistema prefiere escalar a humano antes que inventar. Una venta perdida por escalado es recuperable; una venta con info incorrecta daña la reputación del merchant para siempre.

Marketplace13 / 22

Cross-tenant lead sharing
con commission split

Las tiendas del marketplace ZIAY se refieren clientes entre sí. Si no tienes lo que un cliente busca, lo buscas en otra tienda aliada y ganas comisión. Si otra no lo tiene y tú sí, recibes el lead.

📤 Flujo de outbound (tú refieres)

Cliente pide → No tienes → Marketplace → Aliada despacha → +5% comisión

Lead Router busca en el marketplace por nombre, categoría, especificación. Filtra por reputación y stock. Ofrece al cliente con tu marca (white-label).

📥 Flujo de inbound (te refieren)

Otra tienda → No tiene → Marketplace → Tú sí tienes → Venta −5%

Recibes el lead con todo el contexto (qué pidió, perfil, país). Tu IA lo atiende como si fuera tuyo. Vendes, pagas 5% al que refirió.

💎 Network effect

Más tiendas conectadas →

más opciones para tus clientes (menos "no lo manejamos").

Más opciones →

más ventas cross-tenant (más comisiones para todos).

Más ventas →

más tiendas se conectan (efecto red clásico).

💰 Para el equipo: ojo con esto

El marketplace es el mayor diferenciador competitivo de ZIAY. Ningún competidor lo tiene. Cada mejora al sistema de routing, reputación y conciliación del marketplace multiplica el valor de toda la plataforma.

🧭 Principios del roadmap

1. Métrica antes que feature. Cada nueva capacidad debe mover un KPI de negocio (conversión, AOV, retención). Si no mueve nada, no se lanza.

2. Multi-tenant primero. Todo se construye para escalar a 10.000 tenants sin reescribir. No atajos "solo para un cliente".

3. Compliance no negociable. Cada feature pasa por revisión legal. Si rompe Ley 1581/GDPR, no se lanza — se rediseña.

🧠 Para el equipo

El roadmap no es una lista de deseos — es un contrato con prioridades. Cada feature propuesta se evalúa contra estos 4 quarters. Si no encaja, se pospone. La disciplina de decir "no" es lo que mantiene la calidad.
