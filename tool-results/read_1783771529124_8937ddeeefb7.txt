    83‚Üí### 1.3 Methodology
    84‚Üí
    85‚Üí- **Risk-based sampling for 100% coverage:** every file is assigned a depth tier (Full / Smoke / Existence) based on blast radius; the union of all tiers equals 100% of files.
    86‚Üí- **Evidence-based:** every finding must cite a file:line, a repro command, and a captured artifact under `audit/evidence/`.
    87‚Üí- **Independence:** the Plan agent does not execute findings; execution is delegated to specialized sub-agents (WS-1 ‚Ä¶ WS-12) who cannot approve their own auto-fixes ‚Äî a separate CONSOLIDATE-001 step reviews.
    88‚Üí- **Defense-in-depth:** each control is verified at two layers (code review + runtime probe via agent-browser).
    89‚Üí- **Tooling:** `Read` / `Grep` / `Glob` for static inspection; `agent-browser` skill for end-to-end UI verification; `curl` for API probes; `prisma` CLI for schema introspection; `bun` for runtime checks.

   101‚Üí### 1.5 Audit Principles
   102‚Üí
   103‚Üí1. **Independence:** the auditor does not author the code being audited; auto-fixes are reviewed by a different agent than the one that applied them.
   104‚Üí2. **Evidence-based:** no finding without a reproducible artifact (file:line + screenshot/JSON/log).
   105‚Üí3. **Risk-based:** depth of inspection scales with blast radius (fintech > webhook > catalog adapter > shadcn primitive).
   106‚Üí4. **Least privilege:** audit credentials are read-only; auto-fixes operate on a feature branch, never `main`.
   107‚Üí5. **Reproducibility:** every finding must include the exact command to reproduce.
   108‚Üí6. **Non-regression:** auto-fixes must not break existing tests; if no tests exist, the auditor adds a smoke test before fixing.
   109‚Üí7. **Transparency:** all findings (including rejected ones) are persisted in `audit/findings/`.
   110‚Üí
   111‚Üí### 1.6 Acceptance Criteria for "Production-Ready"
   112‚Üí
   113‚ÜíThe system is **production-ready** when ALL of the following are true:
   114‚Üí
   115‚Üí- 0 Critical findings open
   116‚Üí- 0 High findings open
   117‚Üí- 100% of files inspected (Full + Smoke + Existence tiers complete)
   118‚Üí- 100% of features exercised in browser via agent-browser (14 modules + 2 SSR pages + 1 webhook round-trip)
   119‚Üí- 100% of API routes probed with at least one happy-path + one 401/403/400 case
   120‚Üí- All eligible auto-fixes applied and verified by `tsc --noEmit` + `eslint` + `next build`
   121‚Üí- `AUDIT-REPORT.md` published with executive summary, risk heat-map, and sign-off block
   122‚Üí- Health endpoint reports 0 errors (warnings acceptable if documented)

   556‚Üí
   557‚ÜíFor **every** of the 44 API routes:

   560‚Üí2. Check the route has an auth check (session token, role guard).
   561‚Üí3. Verify the role is allowed per matrix above.
   562‚Üí4. Verify tenantId scope is enforced (cross-tenant access blocked).
   563‚Üí5. Verify the response payload does not leak fields outside the role's read scope.

   571‚Üí| `GET /api/agents` | No auth; lists all 26 agents (low risk ‚Äî metadata only) | Medium |
