  6208в†’### Critical Gaps (Top 10, prioritized)
  6209в†’
  6210в†’**Tier 1 вЂ” Protocol support (blocks the core thesis)**
  6211в†’
  6212в†’1. **AP2 mandate chain (Intent + Cart + Payment) as W3C Verifiable Credentials вЂ” entirely missing.** This is the cryptographic trust primitive that the entire "comercio agГ©ntico" thesis rests on. Without it, ZIAY cannot prove "the user authorized this purchase" in a non-repudiable way, and cannot integrate with Stripe/Adyen's AP2 dashboard support. **Impact:** Cannot sell to enterprises that require auditable AI purchasing. **Fix size:** New Prisma models (`IntentMandate`, `CartMandate`, `PaymentMandate` with `proof`, `issuer`, `holder`, `credentialSchema`, `revoked` columns) + signing service (ed25519 or JSON Web Signature) + 3 API routes + verification middleware in payment flow.
  6213в†’
  6214в†’2. **UCP `/.well-known/ucp` manifest + 4 capabilities вЂ” entirely missing.** External AI agents (Gemini, ChatGPT) cannot discover ZIAY merchants. **Impact:** Invisible to AI shopping discovery; cannot be a UCP-adopting merchant. **Fix size:** Public Next.js route at `src/app/.well-known/ucp/route.ts` returning the JSON manifest from `Tenant` + `payment-registry`; new `/api/ucp/v1/checkout` endpoint implementing the state machine; new `/api/ucp/v1/identity-linking`, `/order`, `/payment-token-exchange`.
  6215в†’
  6216в†’3. **UCP checkout state machine (`incomplete` в†’ `requires_escalation` в†’ `ready_for_complete`) вЂ” missing.** Current `Order.status` is operational, not protocol-level. **Impact:** Cannot signal to an external agent that a human must take over, and cannot hand back a continuation URL. **Fix size:** New `UcpCheckoutSession` model + 3-state transition logic in a new `ucpCheckout.service.ts`; expose continuation URL in `requires_escalation` response.
  6217в†’
  6218в†’4. **ACP (OpenAI/Stripe) support вЂ” entirely missing.** Blocks any future ChatGPT/Copilot Instant Checkout integration. **Impact:** Cannot receive buyers from the OpenAI ecosystem. **Fix size:** Separate `/.well-known/acp`-style manifest (per OpenAI/Stripe spec) + ACP checkout endpoint. Lower priority than UCP because ACP's in-chat checkout was deprioritized by OpenAI in March 2026 (per document В§9.1).
  6219в†’
  6220в†’5. **MCP transport + A2A agent-card.json вЂ” entirely missing.** Claude/ChatGPT cannot call ZIAY as a native tool; no agent-to-agent discovery. **Impact:** Cannot participate in MCP-native agent ecosystems. **Fix size:** New `/api/mcp` route exposing the 4 UCP capabilities as MCP tool definitions; new `/.well-known/agent-card.json` static route.
  6221в†’
  6222в†’**Tier 2 вЂ” Regulatory compliance (blocks Colombia launch)**
  6223в†’
  6224в†’6. **Ley 2573 de 2026 вЂ” traceable identity verification for credit/installment purchases вЂ” entirely missing.** General regime enters force November 2026. **Impact:** Cannot offer credit/installments (cuotas) in Colombia without facing "carga dinГЎmica de la prueba" вЂ” customer is exonerated and ZIAY assumes 100% of the loss on any impersonation claim. **Fix size:** `IdentityVerification` model (documentType, documentNumber, verificationMethod, verificationTimestamp, evidenceHash, verifiedBy) + KYC adapter (could integrate Colombian KYC providers) + gate on `paymentMode` в€€ {credit, installment} routes. AP2 Intent Mandates from gap #1 are explicitly called out in В§12.1 as candidate audit evidence вЂ” they would partially satisfy this.
  6225в†’
  6226в†’7. **Ley 1581 de 2012 вЂ” legal basis tagging + consent revocation mechanism вЂ” entirely missing.** Reform bill Aug 2025 raises fines to 5% of annual revenue. **Impact:** Operating on customer PII (preferences, history, saved payment methods) fed to AI agents without documented legal basis or revocation mechanism is a direct violation. **Fix size:** `ConsentRecord` model (customerId, purpose, legalBasis, grantedAt, revokedAt, source) + `/api/consent` CRUD + `/api/dsr` (data subject request) endpoint + per-column legal-basis annotation in schema docs + agent-runtime consent check before reading PII.
  6227в†’
  6228в†’8. **Supervisor mapping (SIC vs SFC vs MinTIC) вЂ” entirely missing.** No documented regulatory mapping. **Impact:** Cannot demonstrate compliance readiness to an acquirer, bank partner, or investor. **Fix size:** Mostly documentation (a `REGULATORY-MAPPING.md` listing each ZIAY flow в†’ responsible supervisor в†’ applicable article), plus an audit-trail tag on `AuditLog.action` indicating which supervisor's record-keeping rule applies.
  6229в†’
  6230в†’**Tier 3 вЂ” Governance features (blocks trust at scale)**
  6231в†’
  6232в†’9. **Spend limits per category + per-category permissions + mandate revocation вЂ” entirely missing.** Document В§11 lists these as the four governance pillars. **Impact:** Customer cannot delegate purchasing with confidence ("buy beauty up to $200, never electronics"); no revocation means a leaked/stolen agent credential cannot be killed without disabling the user. **Fix size:** `SpendLimit` model (tenantId, customerId, category, maxAmount, windowMs) + `Mandate.revokedAt` column + `/api/mandates/[id]/revoke` endpoint + runtime check in `cart_builder` and `checkout` agents. AP2 Intent Mandate (gap #1) is the natural carrier for these limits.
  6233в†’
  6234в†’10. **AuditLog upgrade to Verifiable Intent compatibility вЂ” partial.** Current `AuditLog` is plain DB rows; document В§11 requires "Registro auditable compatible con Verifiable Intent". **Impact:** Audit log would not satisfy Mastercard's Verifiable Intent framework (donated to FIDO Alliance April 2026), so audit evidence would not be portable across the AP2 ecosystem. **Fix size:** Add `proof` (JWS or Data-Integrity proof) + `credentialSchema` columns to `AuditLog`; sign each row at insert time with the tenant's keypair; expose a `/api/audit/[id]/verifiable` endpoint that returns the row as a W3C VC. Reuse the signing service built for gap #1.

  6315в†’### Business Gaps вЂ” Top 10
  6316в†’
  6317в†’1. **WhatsApp inbound message processing is a stub.** `webhooks/whatsapp/route.ts` POST handler only writes raw body to `auditLog` вЂ” it does NOT parse `entry[].changes[].value.messages`, does NOT upsert `Conversation`/`Message`, does NOT emit via socket.io. Production WhatsApp is non-functional: the agent can never receive a customer message. (Study В§13.1, В§13.6: "WhatsApp: catГЎlogo navegable + agente de IA con tarea concreta".)
  6318в†’
  6319в†’2. **WhatsApp outbound message sending not implemented.** No `POST /{phoneNumberId}/messages` (Cloud API) call anywhere in `src/`. `conversationService.sendMessage()` only persists a local `Message` row. The agent can compose a reply but never actually deliver it to the customer's WhatsApp. Combined with gap #1, the WhatsApp channel is end-to-end broken in production.
  6320в†’
  6321в†’3. **CAPI closed-loop attribution is open.** `capi-fire` job exists and posts to Meta/GA4/TikTok, but the only caller is the explicit `POST /api/conversions` route вЂ” **nothing auto-fires a `Purchase` event when a payment webhook marks an order `paid`**. The study (В§14.4) calls this "the highest-impact improvement reported in 2026" for WhatsApp attribution.
  6322в†’
  6323в†’4. **CTWA `click_id` capture is missing.** Schema fields exist (`Order.clickId`, `sourceAdId`, `sourceCampaign`, `sourcePlatform`, `Attribution`), but because the WA webhook doesn't parse inbound messages, the `ctwa_click_id` Meta attaches to each Click-to-WhatsApp lead is never extracted or persisted. The attribution infrastructure is built but never fed real data.
  6324в†’
  6325в†’5. **First response time (TTR) is not tracked.** No `firstReplyAt` field on `Conversation`; no service computes `min(outbound.createdAt) - min(inbound.createdAt)`. Study В§14.4: "conversion drops >50% if no response in 15вЂ“30 min". Pilot doc sets `<5s` target. **The single most important WhatsApp metric is unmeasurable today.**
  6326в†’
  6327в†’6. **Net contribution margin per channel is not computed.** `channelSplit` shows `{orders, revenue}` per channel but no spend/cost/margin. The central question of В§14 вЂ” "does WhatsApp pay for itself vs. Instagram?" вЂ” cannot be answered. Platform message cost (post-Oct 2026), AI token cost, and human-support cost are all untracked.
  6328в†’
  6329в†’7. **Vertical strategy mismatches study recommendation.** All 4 ZIAY tenants are moda/pijamas + sublimados вЂ” neither is in the study's high-recurrence set (reposiciГіn, repuestos, consumo corporativo, belleza, farmacia, alimentos). Agent-driven repurchase (the core agentic-commerce value prop) is structurally weak in fashion vs. e.g., pharmacy or B2B supplies.
  6330в†’
  6331в†’8. **Multi-country expansion infrastructure is missing.** No multi-currency (COP hardcoded in 6 adapters), no `pt-BR` locale (Brazil blocked), no local payment methods (PSE for CO, PIX for BR, OXXO for MX), no IVA/ISR tax handling. Study В§18 says LATAM is "the next logical layer" вЂ” the codebase cannot serve any market outside Colombia today.
  6332в†’
  6333в†’9. **Trust & safety core requirements are absent.** No spend limits per category, no mandate revocation, no liability definition for out-of-scope purchases вЂ” all explicitly required by В§11. Human escalation exists only as an LLM-decided intent (`escalar_humano`), not as hard rules. AuditLog table exists but decision traceability is partial (`aiSuggested`/`aiConfidence` flags only).
  6334в†’
  6335в†’10. **No pilot/experiment infrastructure.** Zero matches for `pilot|experiment|ab.*test|split.*test` in `src/`. The study (В§14.5) says a real financial model requires "CPL propio medido en un piloto de 4вЂ“6 semanas, tasa de conversiГіn real, margen bruto real por SKU, costo de servicio medido en volumen" вЂ” none of this can be measured in a controlled way today. LTV/retention/churn exist only as LLM prompt text, not as measured DB metrics.
